# 功能说明

## 🎯 核心功能模块

### 1. 文章管理模块

#### 功能描述
提供完整的文章生命周期管理功能，包括创建、编辑、删除、查看和状态管理。

#### 主要功能
- **创建文章**: 支持创建空白文章或从模板创建
- **编辑文章**: Markdown 编辑器，支持实时预览
- **删除文章**: 支持软删除和硬删除
- **文章状态**: 草稿、已发布、已归档
- **文章元数据**: 标题、摘要、封面图、分类、标签

#### 用户故事
```
作为知识工作者
我希望能够快速创建和编辑文章
同时我可以针对文章的部分内容与 ai 进行讨论来激发或者检索更多信息
我可以询问 ai 一些模糊的问题， ai 通过检索相关的文档来给我提供相关的资料和一个概述
我可以收集各种类型的材料放入到我的单个文档中，以扩展我的内容形式，不单纯局限于文本


作为内容创作者
我希望可以设置文章的状态
以便控制哪些内容可以公开发布
```

#### 优先级
- **高**: 创建、编辑、删除文章
- **中**: 文章状态管理、封面图
- **低**: 文章版本历史

---

### 2. 分类和标签模块

#### 功能描述
提供多维度知识组织方式，支持树形分类和扁平化标签。

#### 主要功能
- **分类管理**: 创建、编辑、删除分类，支持多级嵌套
- **标签管理**: 创建、编辑、删除标签，支持颜色标记
- **关联管理**: 为文章指定分类和标签
- **筛选浏览**: 按分类或标签筛选文章列表

#### 用户故事
```
作为用户
我希望可以将文章按主题分类
以便更好地组织我的知识体系

作为用户
我希望可以为文章添加多个标签
以便从不同维度快速找到相关内容
```

#### 优先级
- **高**: 分类管理、标签管理
- **中**: 标签颜色、多级分类
- **低**: 分类拖拽排序

---

### 3. 智能搜索模块

#### 功能描述
基于关键词和语义的混合搜索，提供精准的知识检索能力。

#### 主要功能
- **关键词搜索**: 传统的全文检索
- **语义搜索**: 基于 Embedding 的向量检索
- **混合搜索**: 结合关键词和语义的优势
- **搜索建议**: 自动完成和相关搜索推荐
- **搜索历史**: 记录最近的搜索查询
- **结果高亮**: 高亮显示匹配的关键词

#### 用户故事
```
作为用户
我希望可以通过语义搜索找到相关文章
即使没有完全匹配的关键词

作为用户
我希望可以看到搜索结果的匹配度评分
以便判断最相关的内容
```

#### 优先级
- **高**: 关键词搜索、语义搜索
- **中**: 搜索历史、搜索建议
- **低**: 搜索结果排序算法优化

---

### 4. Markdown 编辑器模块

#### 功能描述
提供所见即所得的 Markdown 编辑体验，支持丰富的语法扩展。

#### 主要功能
- **实时预览**: 左右分栏或即时渲染
- **语法高亮**: 代码块语言识别和高亮
- **数学公式**: LaTeX 公式渲染
- **快捷键**: 常用 Markdown 语法快捷输入
- **图片管理**: 上传、插入、预览图片
- **工具栏**: 常用格式快速插入

#### 用户故事
```
作为写作者
我希望可以使用 Markdown 编写文章
以便专注于内容而不用操心格式

作为开发者
我希望可以插入代码块和数学公式
以便记录技术知识和数学推导
```

#### 优先级
- **高**: 实时预览、语法高亮
- **中**: 数学公式、快捷键
- **低**: 自定义主题、编辑器插件

---

### 5. AI 问答模块（RAG）

#### 功能描述
基于用户的知识库内容，提供智能问答能力。

#### 主要功能
- **提问**: 用户输入自然语言问题
- **检索**: 从知识库中检索相关文章片段
- **生成**: AI 基于检索内容生成答案
- **引用**: 显示答案引用的文章来源
- **多轮对话**: 支持上下文连续对话
- **对话管理**: 查看、清除对话历史

#### 用户故事
```
作为用户
我希望可以向 AI 提问我的知识库内容
以便快速获取相关信息而不用手动查找

作为用户
我希望 AI 回答会引用文章来源
以便验证答案的准确性并深入了解
```

#### 优先级
- **高**: 提问、检索、生成、引用
- **中**: 多轮对话、对话历史
- **低**: 对话分享、导出

---

### 6. AI 辅助编辑模块

#### 功能描述
提供智能化的内容编辑辅助工具。

#### 主要功能
- **内容润色**: 改善措辞、优化结构、修正语法
- **生成摘要**: 自动提取文章核心内容
- **内容扩写**: 根据简要内容扩展成完整文章
- **提取关键词**: 自动识别文章的关键词和标签
- **标题建议**: 生成吸引人的标题建议（可选）

#### 用户故事
```
作为写作者
我希望 AI 可以帮我润色文章
以便提升内容质量

作为用户
我希望可以自动生成文章摘要
以便快速了解文章内容
```

#### 优先级
- **高**: 内容润色、生成摘要、提取关键词
- **中**: 内容扩写
- **低**: 标题建议、翻译功能

---

## 📊 功能依赖关系

```
文章管理 (核心)
    ↓
分类和标签 ────┐
    ↓          │
    └──────────┴──→ 智能搜索
                        ↓
                    AI 问答
                        ↑
                    AI 辅助编辑
```

**说明**:
- 文章管理是核心模块，其他功能都依赖它
- 分类和标签是文章的元数据，用于组织
- 搜索需要索引文章内容
- AI 问答需要检索文章内容
- AI 辅助编辑作用于文章内容

## 🎨 UI/UX 考虑

### 布局设计
- **左侧边栏**: 分类和标签导航
- **中间**: 文章列表或搜索结果
- **右侧**: 文章编辑器或详情
- **浮动**: AI 问答对话框

### 交互设计
- **快速操作**: 键盘快捷键支持
- **拖拽**: 文章拖拽到分类
- **实时反馈**: 保存状态提示
- **渐进式披露**: 高级功能隐藏在菜单中
