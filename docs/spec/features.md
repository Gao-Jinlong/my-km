# 功能说明

## 🎯 核心功能模块

### 0. 项目管理模块 ⭐ 核心创新

#### 功能描述
以"项目"为单元组织知识，支持系统化的知识生产和管理。

#### 主要功能
- **项目创建**: 创建空白项目或从模板创建
- **项目组织**: 一个项目可以包含多篇文章（类似书籍章节）
- **灵活模式**:
  - 多文章模式：多个章节，系统化探索一个主题
  - 单文章模式：一篇文章，凝练核心输出（如技术博客）
- **外部参考资料**: 为项目添加引用模块（PDF、链接、视频等）
- **项目状态**: 草稿、进行中、已完成、已发布
- **项目元数据**: 标题、描述、封面图、标签

#### 用户故事
```
作为技术写作者
我希望可以以项目形式管理我的知识创作
同一个主题下可以有多个章节
也可以只有一篇文章用于快速发布

作为研究员
我希望可以为我研究的主题收集外部参考资料
包括论文、博客、视频等
并可以在项目中快速引用这些资料
```

#### 数据模型设计
```
Project (项目)
├── id, title, description
├── status: DRAFT | IN_PROGRESS | COMPLETED | PUBLISHED
├── articles: Article[] (多篇文章)
├── references: Reference[] (外部参考资料)
└── conversations: Conversation[] (AI 对话历史)

Article (文章)
├── id, title, content
├── projectId: String (关联到项目)
├── order: Int (章节顺序)
└── ...

Reference (参考资料)
├── id, title, url
├── type: LINK | PDF | IMAGE | VIDEO
├── projects: Project[] (可以被多个项目引用)
└── ...
```

#### 优先级
- **v1.0 高**: 项目 CRUD、文章关联、基础参考资料管理
- **v1.1 中**: 项目模板、引用模块增强
- **v2.0 低**: 项目可视化、项目看板

### 1. 文章管理模块

#### 功能描述
提供完整的文章生命周期管理功能，包括创建、编辑、删除、查看和状态管理。

#### 主要功能
- **创建文章**:
  - 独立文章（不属于任何项目）
  - 项目内文章（属于某个项目）
- **编辑文章**: Markdown 编辑器，支持实时预览
- **删除文章**: 支持软删除和硬删除
- **文章状态**: 草稿、已发布、已归档
- **文章元数据**: 标题、摘要、封面图、标签

#### 用户故事
```
作为知识工作者
我希望能够快速创建和编辑文章
无论是独立文章还是项目内的章节

同时我可以针对文章的部分内容与 AI 进行讨论
来激发想法或检索更多信息

我可以询问 AI 一些模糊的问题
AI 通过检索相关的文档来给我提供相关的资料和概述
```

#### 优先级
- **v1.0 高**: 创建、编辑、删除文章、项目关联
- **v1.1 中**: 文章状态管理、封面图
- **v2.0 低**: 文章版本历史

### 2. 引用系统模块 🔗 核心创新

#### 功能描述
所有可编辑单元都可以相互引用，构建知识网络。

#### 主要功能
- **文章级引用**:
  - 文章 → 文章（相关阅读）
  - 文章 → 项目（所属或依赖）
  - 项目 → 项目（前置/后续）
- **双向链接**: 使用 `[[文章标题]]` 语法快速引用
- **反向引用**: 自动显示"所有引用了本文的内容"
- **引用图**: 可视化展示引用关系（v2.0）

#### 用户故事
```
作为用户
我希望可以在文章中引用其他相关文章
以便构建知识网络

作为用户
我希望可以看到哪些内容引用了当前文章
以便了解知识的关联和影响

作为用户
我希望可以看到项目的引用关系图
以便直观理解知识结构
```

#### 优先级
- **v1.0 高**: 文章级引用、双向链接
- **v1.2 中**: 段落级引用
- **v2.0 高**: 引用图可视化、知识图谱

### 3. 智能搜索模块

#### 功能描述
基于关键词和语义的混合搜索，提供精准的知识检索能力。

#### 主要功能
- **关键词搜索**: 传统的全文检索
- **语义搜索**: 基于 Embedding 的向量检索
- **混合搜索**: 结合关键词和语义的优势
- **智能问答式搜索**: 自然语言问题，AI 返回答案+来源
- **搜索建议**: 自动完成和相关搜索推荐
- **搜索历史**: 记录最近的搜索查询
- **结果高亮**: 高亮显示匹配的关键词

#### 用户故事
```
作为用户
我希望可以通过语义搜索找到相关文章
即使没有完全匹配的关键词

例如输入："我之前写过关于微服务的文章吗？"
AI 能返回相关文章列表，而不仅仅是关键词匹配
```

#### 优先级
- **v1.0 高**: 关键词搜索
- **v1.1 高**: 语义搜索、混合搜索
- **v1.2 中**: 搜索历史、智能问答式搜索
- **v2.0 低**: 搜索结果排序算法优化

#### 功能描述

#### 主要功能

#### 用户故事
```
作为写作者

以便专注于内容而不用操心格式

作为开发者
我希望可以插入代码块和数学公式
以便记录技术知识和数学推导
```

#### 优先级
- **v1.0 高**: 实时预览、语法高亮
- **v1.1 中**: AI 快捷操作、数学公式
- **v1.2 中**: 快捷键、工具栏
- **v2.0 低**: 自定义主题、编辑器插件

### 5. AI 深度对话探索模块 🤖 核心创新

#### 功能描述
不仅是"问答"，而是"深度对话探索"，帮助用户梳理观点、凝练草稿。

#### 主要功能

**场景 1: 深度对话 + 凝练草稿**
- 用户输入：我想写关于 X 的文章
- AI 响应：
  1. 检索知识库中的相关内容
  2. 读取用户提供的参考资料
  3. （可选）网络搜索最新资料
  4. 提出探讨问题或建议文章结构
  5. 多轮对话，深入探讨
  6. 最终凝练为文章草稿
- 草稿特性：
  - 标注引用来源（知识库/外部链接/搜索结果）
  - 用户可以编辑、补充、删除
  - 支持从草稿创建正式文章

**场景 2: 材料分析 + 逻辑梳理**
- 用户输入：多个资料（PDF、博客、笔记等）
- AI 响应：
  1. 向量化并分析所有材料
  2. 提取核心主题
  3. 识别材料间的关联性
  4. 构建逻辑关系图
  5. 指出知识缺口
  6. 生成分析报告

#### 用户故事
```
作为技术写作者
我希望在写文章前先和 AI 深度探讨
AI 可以基于我的知识库和外部资料
帮助我梳理观点、搭建框架
最后凝练为文章草稿

作为研究员
我希望 AI 可以帮我分析收集的资料
提取材料中的相关性和内在逻辑
帮助我快速理解研究主题
```

#### 技术流程
```
用户输入主题
    ↓
┌─────────────────────────────┐
│  1. 检索知识库               │
│     - 向量检索相关文章        │
│     - 提取关键片段           │
└──────────┬──────────────────┘
           │
┌──────────▼──────────────────┐
│  2. 读取参考资料             │
│     - 用户提供的链接         │
│     - 项目中的引用模块       │
└──────────┬──────────────────┘
           │
┌──────────▼──────────────────┐
│  3. 网络搜索（可选）         │
│     - Tavily API / SerpAPI  │
│     - 获取最新资料           │
└──────────┬──────────────────┘
           │
┌──────────▼──────────────────┐
│  4. 多轮对话                 │
│     - AI 提出问题            │
│     - 用户回应               │
│     - 累积上下文             │
└──────────┬──────────────────┘
           │
┌──────────▼──────────────────┐
│  5. 凝练草稿                 │
│     - 总结对话结论           │
│     - 标注引用来源           │
│     - 生成结构化草稿         │
└─────────────────────────────┘
```

#### UI 交互设计
- **分屏对话**:
  - 左侧：对话区域
  - 右侧：实时生成的草稿（可编辑）
- **引用标注**:
  - 草稿中高亮显示引用来源
  - 点击可查看原始资料
- **用户控制**:
  - 用户可以随时停止对话
  - 可以要求 AI 调整方向
  - 可以手动编辑草稿

#### 优先级
- **v1.1 高**: 深度对话 + 草稿生成、知识库检索
- **v1.1 中**: 外部链接读取
- **v1.2 高**: 网络搜索集成
- **v1.2 中**: 材料分析和逻辑梳理
- **v2.0 低**: 多模态材料分析（图片、视频）

### 6. AI 辅助编辑模块

#### 功能描述
提供智能化的内容编辑辅助工具，重点是**章节/段落级优化**。

#### 主要功能
- **章节级优化**:
  - 选中一个章节
  - AI 优化该章节，保持与全文连贯
  - 支持润色、扩写、缩写
- **段落级优化**:
  - 选中一个段落
  - AI 优化该段落
  - 考虑前后文语境
- **全文润色**: 改善措辞、优化结构、修正语法
- **生成摘要**: 自动提取文章核心内容
- **提取关键词**: 自动识别文章的关键词和标签
- **连贯性检查**: 检查段落间的逻辑连贯性

#### 用户故事
```
作为写作者
我希望 AI 可以帮我优化单个章节
同时保持与全文的连贯性
而不是每次都重写整篇文章

作为用户
我希望 AI 可以检查文章的逻辑连贯性
指出前后矛盾的地方
```

#### 技术实现
```typescript
// 伪代码：段落级优化
const optimizeParagraph = (targetParagraph, context) => {
  const prompt = `
    上下文：
    前文：${context.beforeParagraphs}
    后文：${context.afterParagraphs}
    文章大纲：${context.articleOutline}

    目标段落：
    ${targetParagraph}

    请优化目标段落，确保：
    1. 与上下文连贯
    2. 符合文章整体风格
    3. 保持原意不变
  `

  return llm.generate(prompt)
}
```

#### 优先级
- **v1.1 高**: 章节/段落级优化、全文润色
- **v1.1 中**: 连贯性检查
- **v1.2 中**: 生成摘要、提取关键词
- **v2.0 低**: 内容扩写、标题建议

### 7. 知识分享模块

#### 功能描述
从"个人知识库"到"职业价值展示平台"，支持用户公开分享知识。

#### 主要功能
- **用户主页**:
  - 个人介绍
  - 知识项目列表（公开）
  - 独立文章列表（公开）
- **权限管理**:
  - 私有：仅自己可见
  - 公开：所有人可访问
- **项目展示**:
  - 项目介绍
  - 章节列表
  - 参考资料（可选）
- **分享功能**:
  - 生成分享链接
  - 社交媒体分享按钮
  - （未来）评论和反馈

#### 用户故事
```
作为技术写作者
我希望可以将我的知识项目公开分享
以便建立个人品牌和展示专业能力

作为用户
我希望可以查看其他用户的知识项目
以便学习和交流
```

#### 优先级
- **v1.0 高**: 用户主页、项目/文章公开设置
- **v1.1 中**: 分享链接
- **v3.0 低**: 评论和反馈

## 📊 功能依赖关系

```
项目管理 (核心)
    ↓
文章管理 ──────┐
    ↓          │
引用系统 ──────┤
    ↓          │
参考资料管理 ──┤
    ↓          │
    └──────────┴──→ 智能搜索
                        ↓
                    AI 深度对话
                        ↑
                    AI 辅助编辑
                        ↑
                    知识分享
```

**说明**:
- 项目管理是核心组织单元
- 文章管理是内容载体
- 引用系统连接知识单元
- AI 能力贯穿整个工作流
- 知识分享是最终价值出口

## 🎨 UI/UX 考虑

### 布局设计
- **项目视图**:
  - 左侧边栏：项目列表和导航
  - 中间：文章树形结构（章节）
  - 右侧：当前文章编辑器
  - 底部/浮层：参考资料 + AI 对话
- **文章视图**:
  - 左侧边栏：分类和标签导航
  - 中间：文章列表或搜索结果
  - 右侧：文章编辑器或详情
  - 浮动：AI 对话对话框

### 交互设计
- **快速操作**: 键盘快捷键支持
- **拖拽**: 文章拖拽到项目
- **实时反馈**: 保存状态提示、AI 生成进度
- **渐进式披露**: 高级功能隐藏在菜单中
