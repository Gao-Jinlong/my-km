# 执行计划 (Execution Plans)

> 追踪长期、跨 PR 的任务和计划

## 📋 什么是执行计划

执行计划用于追踪需要多个 PR 迭代完成的长期任务。每个执行计划包含：
- 明确的目标和范围
- 分阶段的任务列表
- 进度追踪
- 相关 PR 链接

## 📁 目录结构

```
exec-plan/
├── README.md           # 本文件
└── <plan-name>.md      # 具体的执行计划文档
```

## 📝 执行计划模板

```markdown
# <计划名称>

**状态**: 🟡 进行中 / 🟢 已完成 / 🔴 已暂停
**创建日期**: YYYY-MM-DD
**负责人**: @xxx

## 目标

简要描述该计划要达成的目标。

## 任务列表

- [ ] 任务 1 - PR #xxx
- [ ] 任务 2 - PR #xxx
- [ ] 任务 3 - PR #xxx

## 进度

| 阶段 | 任务 | 状态 | PR |
|------|------|------|-----|
| 1 | 任务描述 | ✅ | #123 |
| 2 | 任务描述 | 🟡 | - |

## 相关文档

- [相关文档链接]()
```

## 🔄 与 OpenSpec 的关系

| 工具 | 用途 | 范围 |
|------|------|------|
| **OpenSpec** | 单个 PR 的需求迭代 | 短期、单一变更 |
| **exec-plan** | 跨 PR 的长期任务追踪 | 长期、多阶段任务 |
