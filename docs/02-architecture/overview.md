# 系统架构概览

## 项目概述

**my-km** 是一个 AI 原生的个人知识工作室，基于 Next.js 16 + NestJS 11 的全栈 Monorepo 项目。

### 核心特性
- **项目化管理**: 用户选择本地文件夹作为项目，在授权范围内进行文件操作
- AI 深度对话探索
- 智能引用网络
- 知识分享平台

### 重要说明

> 受浏览器 File System Access API 限制，系统**不能**在用户文件系统中直接创建文件夹。
> 用户需要先在本地创建一个空文件夹，然后通过"打开文件夹"功能选择它。

## Monorepo 结构

```
my-km/
├── apps/
│   ├── web/          # Next.js 前端 (:4000)
│   └── server/       # NestJS 后端 (:3001)
├── packages/
│   ├── prisma/       # Prisma ORM
│   └── shared/       # 共享代码
├── docs/             # 项目文档
└── openspec/         # 变更管理
```

## 前端架构

```
┌─────────────────────────────────────────┐
│            UI 组件层                      │
├─────────────────────────────────────────┤
│          Provider 层                     │
├─────────────────────────────────────────┤
│          Service 层                      │
│  - CommandRegistry  - MenuRegistry       │
│  - ConditionService - FileManager        │
├─────────────────────────────────────────┤
│          Platform 层 (平台抽象)            │
└─────────────────────────────────────────┘
```

## 后端架构

```
┌─────────────────────────────────────────┐
│            API 层                         │
├─────────────────────────────────────────┤
│          Service 层                      │
│  - Auth  - User  - Cache  - Email        │
├─────────────────────────────────────────┤
│          Data 层 (Prisma + PostgreSQL)   │
└─────────────────────────────────────────┘
```
