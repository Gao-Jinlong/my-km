# 需求文档

## 📋 用户需求分析

### 目标用户
- 知识工作者（开发者、研究员、写作者等）
- 需要长期积累和管理知识的个人用户
- 希望通过 AI 提升知识管理效率的用户

### 核心痛点
1. **知识分散**: 文档散落在各个平台，难以统一管理
2. **检索困难**: 传统的关键词搜索无法满足语义检索需求
3. **整理耗时**: 手动分类和标签需要大量时间
4. **利用不足**: 已积累的知识难以有效复用和关联

---

## 🎯 功能需求

### 1. 文章管理
- **FR-1.1**: 用户可以创建、编辑、删除文章
- **FR-1.2**: 支持 Markdown 格式编写
- **FR-1.3**: 支持设置文章状态（草稿、已发布、已归档）
- **FR-1.4**: 支持添加文章摘要和封面图
- **FR-1.5**: 支持文章版本历史（可选）

### 2. 分类和标签
- **FR-2.1**: 支持树形分类结构
- **FR-2.2**: 支持多级分类嵌套
- **FR-2.3**: 支持为文章添加多个标签
- **FR-2.4**: 支持标签颜色自定义
- **FR-2.5**: 支持按分类和标签筛选文章

### 3. 智能搜索
- **FR-3.1**: 支持全文关键词搜索
- **FR-3.2**: 支持语义搜索（基于 Embedding）
- **FR-3.3**: 支持混合搜索（关键词 + 语义）
- **FR-3.4**: 显示搜索结果的相关性评分
- **FR-3.5**: 支持搜索历史记录
- **FR-3.6**: 提供搜索建议和自动完成

### 4. Markdown 编辑器
- **FR-4.1**: 实时预览 Markdown 渲染效果
- **FR-4.2**: 支持代码高亮
- **FR-4.3**: 支持数学公式渲染
- **FR-4.4**: 支持快捷键编辑
- **FR-4.5**: 支持图片上传和插入

### 5. AI 问答（RAG）
- **FR-5.1**: 用户可以向 AI 提问
- **FR-5.2**: AI 基于用户的知识库回答问题
- **FR-5.3**: 显示回答引用的文章来源
- **FR-5.4**: 支持多轮对话
- **FR-5.5**: 支持对话历史管理

### 6. AI 辅助编辑
- **FR-6.1**: 内容润色（改善措辞和结构）
- **FR-6.2**: 生成摘要
- **FR-6.3**: 内容扩写
- **FR-6.4**: 自动提取关键词
- **FR-6.5**: 生成标题建议（可选）

## 🔒 非功能性需求

### 性能要求
- **NFR-1**: 文章列表加载时间 < 500ms
- **NFR-2**: 搜索响应时间 < 1s
- **NFR-3**: AI 问答首字响应时间 < 2s
- **NFR-4**: 支持至少 10,000 篇文章

### 可用性要求
- **NFR-5**: 界面简洁直观，学习成本低
- **NFR-6**: 支持键盘快捷键操作
- **NFR-7**: 移动端友好（响应式设计）

### 可维护性要求
- **NFR-8**: 代码模块化，易于扩展
- **NFR-9**: 完善的 API 文档
- **NFR-10**: 数据库迁移脚本版本化

### 安全性要求
- **NFR-11**: SQL 注入防护
- **NFR-12**: XSS 攻击防护
- **NFR-13**: API 请求速率限制（未来）

### 兼容性要求
- **NFR-14**: 支持现代浏览器（Chrome, Firefox, Safari, Edge）
- **NFR-15**: 支持 PostgreSQL 15+
- **NFR-16**: 支持 Node.js 20+

## 🚫 暂不实现的功能

### 当前版本不考虑
- 多用户支持和权限管理
- 文章分享和公开链接
- 协作编辑
- 文件附件管理
- 导出为 PDF/EPUB
- 数据导入（从其他知识库工具）

### 未来版本可能考虑
- Web Clipper 浏览器插件
- 移动端 App
- 离线模式
- 数据备份和恢复
- 数据同步
