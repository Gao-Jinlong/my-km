DocumentModel 详细设计

文档模型层 - 核心数据结构与接口

1. 核心概念

设计原则：粗粒度块设计
📦 领域模型 (Domain)
Document {
id: string
title: string
blocks: Block[]
version: number
metadata: DocMetadata
}

Block {
id: string
type: BlockType
content: BlockContent
children?: Block[]
metadata: BlockMeta
}
🔌 Lexical 映射 (Platform)
Document → EditorState
Block → Node[]

TextBlock → ParagraphNode + TextNode
HeadingBlock → HeadingNode + TextNode
CodeBlock → CodeNode + CodeLineNode
TableBlock → TableNode + RowNode + CellNode
ImageBlock → ImageNode

2. BlockType 定义

选择需要支持的块类型（基于之前讨论的 7 种）

A

TextBlock (文本块)

段落、标题（通过 level 区分）、引用

{ type: "text", level?: 1|2|3|4|5|6, style?: "normal"|"quote" }
B

ListBlock (列表块)

有序列表、无序列表、待办清单

{ type: "list", listType: "ordered"|"unordered"|"task", checked?: boolean }
C

CodeBlock (代码块)

带语法高亮的代码块

{ type: "code", language: string, content: string }
D

TableBlock (表格块)

行列结构的表格

{ type: "table", rows: number, cols: number, cells: CellData[] }
E

ImageBlock (图片块)

图片及说明文字

{ type: "image", src: string, alt?: string, caption?: string }
F

EquationBlock (公式块)

LaTeX 数学公式

{ type: "equation", latex: string, displayMode: boolean }

3. 序列化格式

JSON 序列化格式示例
{
  "id": "doc-001",
  "title": "未命名文档",
  "version": 1,
  "createdAt": "2026-03-25T10:00:00Z",
  "updatedAt": "2026-03-25T10:30:00Z",
  "blocks": [
    {
      "id": "block-001",
      "type": "text",
      "content": { "text": "这是一个标题", "level": 1 },
      "metadata": { "collapsed": false }
    },
    {
      "id": "block-002",
      "type": "text",
      "content": { "text": "这是正文，支持加粗和斜体" },
      "metadata": {}
    },
    {
      "id": "block-003",
      "type": "code",
      "content": { "language": "typescript", "code": "console.log('Hello')" },
      "metadata": {}
    }
  ]
}

4. 需要确认的设计决策

请确认以下设计是否合理：

  1. 块 ID 生成策略
    • nanoid() - 短小、安全、唯一
    • 前缀区分：block-xxx vs doc-xxx
  2. 块内容的存储方式
    • 方案 A：纯文本 + 标记（轻量，易编辑）
    • 方案 B：Lexical EditorState JSON（完整，保真）
    • 方案 C：混合（简单文本用方案 A，复杂块用方案 B）
  3. 版本管理策略
    • 每次保存递增 version
    • 同时记录 timestamp
    • 未来支持版本历史回溯

提示： 请在浏览器中选择块类型（默认全选），然后在终端告诉我你对设计决策的偏好。
