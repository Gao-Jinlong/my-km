版本管理演进路线

Phase 1: MVP
Document {
  version: number      // 递增版本号
  updatedAt: string    // ISO 时间戳
  snapshot: JSON       // 完整快照
}
✅ 简单可靠 · ❌ 无法回溯历史
→
Phase 2: Operation Log
Document {
  version: number
  operations: Op[]   // 操作日志
  snapshot: JSON     // 基础快照
}

Operation {
  id: string
  type: "insert" | "delete" | "update"
  blockId: string
  data: JSON
  timestamp: string
}
✅ 支持撤销/历史 · 🟡 实现复杂
设计约束： Phase 1 的代码结构必须能平滑升级到 Phase 2，避免重写。 Document 预留 operations?: Operation[] 字段。
