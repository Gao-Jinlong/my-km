BlockRegistry 详细设计

块类型注册中心 - 连接领域模型与 Lexical 的桥梁

1. 核心职责

📦 类型注册

注册所有块类型，维护 BlockType → Config 的映射表

🔄 双向映射

Block ↔ Lexical Node 的序列化/反序列化

🏭 工厂创建

根据类型创建新块实例，支持默认值

2. 接口定义

interface BlockTypeConfig {
  type: string;                    // 块类型标识
  name: string;                    // 人类可读名称
  category: BlockCategory;      // 分类：文本/媒体/高级
  icon: string;                    // 图标（用于菜单）
  description: string;             // 描述（用于斜杠菜单）
  defaultContent: () => JSON;      // 默认内容工厂
  toLexical: (block: Block) => Node;    // Domain → Lexical
  fromLexical: (node: Node) => Block; // Lexical → Domain
  isValid: (content: JSON) => boolean;    // 内容校验
  allowedChildren?: BlockType[];  // 允许的子块类型
}

class BlockRegistry {
  // 注册块类型
  register(config: BlockTypeConfig): void;

  // 获取类型配置
  get(type: string): BlockTypeConfig | undefined;

  // 查询是否已注册
  has(type: string): boolean;

  // 列出所有类型
  list(): BlockTypeConfig[];

  // 按分类筛选
  listByCategory(category: BlockCategory): BlockTypeConfig[];

  // 创建块实例
  createBlock(type: string, overrides?: Partial<Block>): Block;

  // 序列化
  serialize(block: Block): JSON;

  // 反序列化
  deserialize(json: JSON): Block;

  // Lexical 互转
  toLexical(block: Block): Node;
  fromLexical(node: Node): Block;
}

3. 块类型分类

📝 文本类 (Text)
  • paragraph - 段落
  • heading - 标题 (H1-H6)
  • quote - 引用
  • callout - 提示框
📋 列表类 (List)
  • bullet-list - 无序列表
  • numbered-list - 有序列表
  • task-list - 待办清单
🖼️ 媒体类 (Media)
  • image - 图片
  • code - 代码块
  • table - 表格
  • equation - 公式
  • file-attachment - 附件

4. 注册流程

// 1. 定义块类型配置
const textBlockConfig: BlockTypeConfig = {
  type: 'text',
  name: '文本',
  category: 'text',
  icon: '📝',
  description: '正文内容',

  defaultContent: () => ({
    text: '',
    level: undefined,
    style: 'normal'
  }),

  toLexical: (block) => {
    // Block → Lexical Node 转换逻辑
    const paragraph = ParagraphNode.create();
    const text = TextNode.create(block.content.text);
    paragraph.append(text);
    return paragraph;
  },

  fromLexical: (node) => {
    // Lexical Node → Block 转换逻辑
    if (!(node instanceof ParagraphNode)) {
      throw new Error('Invalid node type');
    }
    return {
      type: 'text',
      content: { text: node.getTextContent() }
    };
  },

  isValid: (content) => typeof content.text === 'string'
};

// 2. 注册到 BlockRegistry
blockRegistry.register(textBlockConfig);
blockRegistry.register(headingBlockConfig);
blockRegistry.register(codeBlockConfig);
// ... 注册所有块类型

6. 设计决策

确认以下设计细节：

A

单例模式

全局唯一 BlockRegistry 实例，应用启动时初始化

B

懒加载

块类型按需加载，首次使用时注册

C

支持插件扩展

暴露 register() API，允许第三方注册自定义块

D

严格校验

反序列化时严格校验内容格式，失败则降级为纯文本
