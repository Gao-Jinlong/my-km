A. Family sections 推荐

按规范 §3.2 家族分组；每组内每个组件一张 compact matrix。最适合权威设计稿和后续查找。

Static · Card / Badge / Kbd / Code / Table
Stateful · Button / Input / Tabs / Accordion
Overlay · Dialog / Drawer / Popover / Menus

B. Interaction density

按交互复杂度组织，利于实现优先级，但会偏离现有 spec 的家族分类。

C. Storybook index
